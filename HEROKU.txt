Heroku: fix "No default language could be detected"

  Heroku looks at the REPO ROOT for composer.json / package.json. This monorepo has
  Laravel only under backend/, so auto-detection fails until buildpacks are set.

  You MUST configure buildpacks (and PROJECT_PATH) on the Heroku app — pushing code
  alone is not enough.

────────────────────────────────────────
A) Heroku Dashboard (easiest)

  1) https://dashboard.heroku.com → select your app → Settings.

  2) Buildpacks → Clear any wrong entries, then Add buildpack IN THIS ORDER:
       First:  https://github.com/timanovsky/subdir-heroku-buildpack
       Second: heroku/php
     (If you can reorder, subdir must be above PHP.)

  3) Settings → Config Vars → Add:
       PROJECT_PATH = backend
     (no trailing slash)

  4) Deploy → Deploy Branch (or push again after saving).

────────────────────────────────────────
B) Heroku CLI

  heroku buildpacks:clear -a YOUR_APP
  heroku buildpacks:set https://github.com/timanovsky/subdir-heroku-buildpack -a YOUR_APP
  heroku buildpacks:add heroku/php -a YOUR_APP
  heroku config:set PROJECT_PATH=backend -a YOUR_APP

────────────────────────────────────────
Also set Laravel env (examples)

  heroku config:set APP_KEY=base64:... APP_ENV=production APP_DEBUG=false -a YOUR_APP
  (Use DATABASE_URL, SANCTUM_*, CORS_*, STRIPE_* from backend/.env.example.)

  backend/Procfile: web server for Laravel public/

  Frontend: deploy on Vercel; set VITE_API_URL to this Heroku app URL.

────────────────────────────────────────
Repo files: app.json and .buildpacks

  Some workflows use these as hints. Heroku may still require Dashboard/CLI buildpacks.
  New app from manifest: heroku create --manifest (reads app.json).
