Heroku deploy (Laravel API)

  The repo root now has composer.json + composer.lock + Procfile so the heroku/php
  buildpack can run without a subdirectory buildpack. Dependencies install into
  backend/vendor; the web process serves backend/public/.

One-time

  heroku stack:set heroku-24 -a YOUR_APP
  heroku buildpacks:set heroku/php -a YOUR_APP

  Set config vars from backend/.env.example (APP_KEY, DATABASE_URL, APP_URL, SANCTUM_*,
  CORS_*, STRIPE_*, etc.).

After changing PHP dependencies

  Prefer updating from the repo root so both lock files stay aligned:

    cd /path/to/repo
    composer update
    cp composer.lock backend/composer.lock

  Or edit backend/composer.json, run composer update in backend/, then copy
  backend/composer.lock to the repo root and merge any require changes into the
  root composer.json (keep autoload paths with the backend/ prefix).

Local development

  You can keep using "cd backend && composer install" as before; keep
  backend/composer.lock in sync with the root lock file when you change deps.

Frontend

  Deploy on Vercel; set VITE_API_URL to your Heroku app URL.

HTTP 500 on Heroku

  - Attach Heroku Postgres so DATABASE_URL is set (Laravel now maps DATABASE_URL → DB URL).
  - Run migrations: heroku run php artisan migrate --app YOUR_APP
  - Ensure APP_KEY is set: heroku config:get APP_KEY
  - Read the error: heroku logs --tail --app YOUR_APP
